

# Источники и факт-чекинг в генерации профайла

## Текущее состояние

### Сбор источников
Сейчас AI получает данные из **двух** источников:
1. **Карточка партнёра** — поля из таблицы `partners` (название, отрасль, сайт, legacy-профиль)
2. **Скрейпинг сайта** — Firecrawl забирает markdown с `website_url` (до 30K символов)

AI сам генерирует список `references` — номера, названия, URL. Но эти ссылки **не верифицируются**: AI может выдумать URL или приписать факт не тому источнику.

### Проблемы
- Нет веб-поиска — AI не ищет новости, финансовые данные, упоминания в прессе
- Нет факт-чекинга — ни один факт не проверяется после генерации
- Ссылки могут быть «галлюцинацией» — AI генерирует красивые URL, которых не существует
- Загруженные файлы (`.docx`, `.pdf`) из `partner-profile-files` **не используются** при генерации

## Предложение: двухпроходная генерация с самопроверкой

### Шаг 1: Расширить базу источников
- **Веб-поиск** через Firecrawl Search или web_search — найти 5-10 релевантных страниц (новости, РБК, ТАСС, сайт компании)
- **Загруженные файлы** — извлечь текст из PDF/DOCX в `partner-profile-files` и передать как контекст
- Каждому источнику присвоить номер **до** генерации, передать AI готовый список

### Шаг 2: Генерация профайла (как сейчас)
AI заполняет секции, ссылаясь на пронумерованные источники `[N]`

### Шаг 3: Самопроверка (второй вызов AI)
Отдельный вызов с промтом:
```
Ты — факт-чекер. Тебе дан профайл и исходные тексты источников.
Для каждого факта с ссылкой [N]:
1. Найди этот факт в тексте источника N
2. Если факт подтверждён — ✅
3. Если факт не найден в источнике — ⚠️ (не подтверждён)
4. Если факт противоречит источнику — ❌ (противоречие)
Верни JSON с результатами проверки по секциям.
```

### Шаг 4: UI — отображение результатов
- Бейдж у каждой секции: `✅ 12/12 фактов подтверждены` или `⚠️ 3 факта не подтверждены`
- При клике — список фактов с цветовой индикацией
- Неподтверждённые факты подсвечены жёлтым в тексте

## Ограничения
- Двойной вызов AI увеличит время генерации (~×2) и расход кредитов
- Веб-поиск требует дополнительных API-вызовов Firecrawl
- Факт-чекинг LLM-ом не абсолютно надёжен — но значительно лучше, чем ничего
