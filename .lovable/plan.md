

# Прикрепление файлов при создании нового элемента портфолио

## Проблема
Сейчас файлы можно прикрепить только при редактировании (когда уже есть `portfolio_item_id`). При создании нового элемента секция «Файлы» не отображается.

## Решение
Изменить логику сохранения: при создании нового элемента — сначала вставить запись в БД, получить `portfolio_item_id`, затем **не закрывать диалог**, а переключиться в режим редактирования этого элемента. Пользователь увидит появившуюся секцию «Файлы» и сможет прикрепить документы. Кнопка изменится на «Сохранить».

### Как это будет работать
1. Пользователь нажимает «Добавить» в диалоге нового элемента
2. Элемент сохраняется в БД, возвращается `portfolio_item_id`
3. Диалог **остаётся открытым**, `editingPortfolioId` устанавливается на новый ID
4. Появляется секция «Файлы» — можно загружать документы
5. Кнопка меняется на «Сохранить», повторное нажатие — обновление + закрытие

### Изменения в `UnitDetail.tsx` и `UnitContactDetail.tsx`
- В `savePortfolio.mutationFn`: при создании (insert) — вернуть `portfolio_item_id` через `.select().single()`
- В `onSuccess`: если это было создание — установить `editingPortfolioId` на новый ID, **не закрывать** диалог, показать toast «Сохранено — можно прикрепить файлы»
- Если это было обновление — закрыть диалог как раньше

### Затронутые файлы
- `src/pages/UnitDetail.tsx`
- `src/pages/UnitContactDetail.tsx`

Без изменений БД.
