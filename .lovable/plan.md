

# Исправление PDF-экспорта: наложение текста и escaped-скобки

## Проблемы в текущем PDF

### Критические (видны на каждой странице)
1. **Текст наложен сам на себя** — каждая строка с bold-фрагментами рендерится дважды: один раз как plain text через `doc.text(wLine)`, второй раз через `drawRichLine(runs)`. Результат — нечитаемая каша из наложенных строк.
2. **`drawRichLine` рисует весь текст целиком** — при word-wrap текст разбивается на несколько строк, но `drawRichLine` всегда рисует ВСЕ runs с начала, а не только текст текущей wrapped-строки. Поэтому на каждой перенесённой строке рисуется полный текст абзаца.
3. **Escaped скобки `\[N\]`** — в таблицах (autoTable) скобки отображаются как `\[3\]` вместо `[3]`. Функция `preprocessText` обрабатывает только текстовые блоки, но не данные таблиц.
4. **Страница 4 почти пустая** — один источник на целой странице, нет попытки уместить блок «Источники» компактнее.

### Корневая причина
Функция `renderTextBlock` неправильно обрабатывает word-wrap для строк с bold-фрагментами:
- `doc.splitTextToSize(plainText, contentW)` разбивает текст на строки
- Но `drawRichLine(runs, ...)` рисует ВСЕ runs, не зная какой кусок текста соответствует текущей wrapped-строке
- В результате каждая wrapped-строка получает полное содержимое, наложенное на правильный текст

## Решение

Полностью переписать логику рендеринга текста с правильной обработкой word-wrap для rich text.

### Новый подход: пословное рисование с переносами

Вместо `splitTextToSize` + `drawRichLine` использовать **пословную отрисовку**:
1. Разбить каждый run на слова, сохраняя атрибут bold
2. Рисовать слова последовательно, отслеживая позицию X
3. Когда слово не помещается — перенос на новую строку
4. Для каждого слова переключать шрифт (bold/normal)

Этот подход гарантирует, что каждое слово рисуется ровно один раз в правильной позиции.

### Дополнительные исправления
- Применять `preprocessText` к данным таблиц перед передачей в autoTable
- Убрать дублирующий вызов `doc.text()` для строк с bold-фрагментами

- Скобки `[N]` будут без обратных слэшей везде, включая таблицы
