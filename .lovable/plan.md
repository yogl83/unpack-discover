

# Улучшение карточек коллективов и компетенций

## Анализ текущего состояния

**Карточка коллектива (UnitDetail):**
- Вкладки: Информация → Компетенции → Гипотезы → Портфолио → Контакты
- Портфолио — только просмотр (нет кнопки «Добавить», нет CRUD)
- Состав коллектива формируется из `unit_contacts` через `unit_contact_memberships` — уже работает
- Описание команды (`team_summary`) — текстовое поле

**Карточка компетенции (CompetencyDetail):**
- Плоская форма без вкладок
- Поле `education_link` — «Ссылка на образование» — действительно нелогично для команды

---

## Предлагаемые изменения

### 1. Портфолио коллектива — полноценный CRUD + перемещение вверх

**Проблема:** Портфолио read-only, нет возможности добавлять/редактировать/удалять элементы.

**Решение:**
- Добавить кнопку «Добавить» в заголовок вкладки Портфолио
- Inline-форма или диалог для создания/редактирования элемента (title, item_type, organization_name, description, year_from, year_to, url, notes)
- Кнопки редактирования и удаления в каждой строке таблицы
- **Переместить вкладку Портфолио на 2-е место** (после Информации), по аналогии с профайлом организации

Типы портфолио для select: `project` (Проект), `publication` (Публикация), `patent` (Патент), `grant` (Грант), `product` (Продукт), `other` (Другое).

### 2. Компетенция — убрать «Ссылка на образование»

**Проблема:** Поле `education_link` не имеет смысла для компетенции команды.

**Решение:**
- Убрать поле из формы (оставить в БД для обратной совместимости, просто не показывать)
- Заменить на более релевантное поле, например ссылку на публикации или ресурсы команды — или просто убрать

### 3. Портфолио коллектива — генерация на основе данных (фаза 2)

По аналогии с профайлом организации, в будущем можно добавить:
- Загрузку файлов-источников
- LLM-генерацию описания портфолио
- Пока что — ручной CRUD как MVP
