

# Добавление в коллектив через выбор из существующих сотрудников

## Проблема
Кнопка «Добавить» в коллективе открывает форму создания нового контакта с нуля. Нужно выбирать из уже существующих сотрудников подразделения и указывать только роль в команде.

## Решение

В `src/pages/UnitDetail.tsx`:

1. **Заменить навигацию на диалог выбора**: вместо `navigate(/units/${id}/contacts/new)` — открывать `Dialog` с:
   - Выпадающий список (`Select`) сотрудников подразделения из `unit_contacts`, исключая тех, кто уже в `memberships`
   - Выпадающий список роли в команде (`member_role`) с вариантами из `memberRoleLabels` (lead, researcher, engineer, student, other)
   - Кнопка «Добавить» вызывает существующий `addMembership` mutation

2. **Состояние диалога**: добавить `showAddMemberDialog` (boolean), использовать уже существующие `addMemberContact` и `addMemberRole` state-переменные

3. **Фильтрация**: показывать только тех `unit_contacts`, чей `unit_contact_id` отсутствует в текущих `memberships`

## Затронутый файл
- `src/pages/UnitDetail.tsx` — ~30 строк: замена кнопки + добавление диалога
