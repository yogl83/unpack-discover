

# Настройка отправки писем

## Текущее состояние
- Письма **не отправляются** при регистрации, одобрении и создании аккаунтов
- Работает только сброс пароля (дефолтный шаблон Supabase Auth)
- Почтовый домен не настроен

## Что нужно сделать

### Шаг 1: Настроить почтовый домен
Для отправки писем с вашего домена нужно настроить email-домен через Cloud. Без этого письма будут уходить только через дефолтные шаблоны.

### Шаг 2: Добавить отправку уведомлений

**При одобрении заявки** (`approve-user`):
- Отправлять пользователю письмо «Ваш доступ подтверждён, вы можете войти в систему»

**При создании пользователя админом** (`create-admin`):
- Отправлять письмо с логином и временным паролем (или ссылкой на установку пароля)

**При подаче заявки на регистрацию** (`register-request`):
- Отправлять подтверждение «Ваша заявка принята, ожидайте одобрения»

### Шаг 3: Кастомизация шаблонов Auth
Настроить брендированные шаблоны для сброса пароля (вместо дефолтных).

## Техническая реализация
- Настройка email-домена через диалог Cloud
- Создание email-инфраструктуры (очереди, Edge Functions)
- 3 транзакционных шаблона (одобрение, создание, заявка)
- Модификация Edge Functions `approve-user`, `create-admin`, `register-request` для вызова `send-transactional-email`
- Опционально: кастомные Auth-шаблоны для сброса пароля

## Вопрос
Нужно ли настраивать отправку с вашего домена, или достаточно чтобы письма уходили с дефолтного адреса Lovable?
