

# Добавить ключевые библиографические поля для РИД

## Проблема
Сейчас у РИД только: название, правообладатель, год регистрации, номер/ссылка, описание. Для полной библиографической ссылки не хватает ключевых полей — **авторы** и **номер регистрации** (отдельно от URL).

## Традиционные поля РИД для библиографии

| Поле | Текущее состояние | Действие |
|------|-------------------|----------|
| Название | ✅ title | — |
| Авторы | ❌ нет | Добавить `authors` (text) |
| Тип РИД | ✅ rid_subtype | — |
| Номер регистрации | ❌ смешан с URL | Добавить `registration_number` (text) |
| Дата регистрации | ✅ year_from | — |
| Правообладатель | ✅ organization_name | — |
| Ссылка | ✅ url | Оставить для внешней ссылки |
| Страна | ❌ нет | Добавить `country` (text, по умолчанию «RU») |

## Изменения

### 1. БД — миграция
Добавить 3 поля в `unit_portfolio_items` и `contact_portfolio_items`:
- `authors` (text) — список авторов
- `registration_number` (text) — номер патента / свидетельства
- `country` (text) — страна регистрации

### 2. `src/lib/labels.ts`
Обновить `portfolioFieldConfig.rid`:
- `urlLabel` → «Ссылка» (номер теперь отдельно)
- `urlPlaceholder` → «https://...»

### 3. UI — `UnitDetail.tsx` и `UnitContactDetail.tsx`
Когда `item_type === 'rid'`, показывать дополнительные поля:
- **Авторы** (текстовое поле, placeholder: «Иванов И.И., Петров П.П.»)
- **Номер регистрации** (input, placeholder: «RU 2 123 456» или «2024615678»)
- **Страна** (input, по умолчанию «RU»)

Добавить эти поля в форму, стейт и логику сохранения.

### Затронутые файлы
- Миграция SQL (3 новых поля × 2 таблицы)
- `src/lib/labels.ts`
- `src/pages/UnitDetail.tsx`
- `src/pages/UnitContactDetail.tsx`
