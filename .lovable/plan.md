

# Финальная полировка PDF-экспорта

## Что исправить

### 1. Таблица конфликтует с футером (стр. 1)
`bodyBottom = pageH - MARGIN - FOOTER_H = 297 - 15 - 10 = 272mm`. Но `FOOTER_H = 10` может быть недостаточно — футер рисуется на `pageH - MARGIN = 282mm`, линия на 278mm. Увеличить `FOOTER_H` с 10 до 14mm, чтобы контент гарантированно не заходил в зону футера.

### 2. Артефакт `\-` в тексте
В `preprocessText` добавить: `.replace(/\\-/g, "-")` — убирает escaped дефис. Также добавить общий `.replace(/\\([^\[\]])/g, "$1")` для остальных escaped символов.

### 3. Увеличить отступ после вводного абзаца
После summary_short увеличить gap: `y += SECTION_GAP` → `y += SECTION_GAP + 4` (с 8 до 12mm).

### 4. Таблица — внутренние отступы
В autoTable увеличить `cellPadding` с 3 до 4, добавить `minCellHeight: 8`.

### 5. Источники — не разрывать посередине
Перед каждым источником вычислять примерную высоту блока (title + url + quotes) и вызывать `ensureSpace` с этой суммой, чтобы источник не разрывался между страницами. Если источник слишком длинный — ставить `currentSectionLabel = "Источники"` для continuation header.

### 6. Источники — усилить визуальную иерархию
- Номер+название: 9pt bold (было 8.5)
- URL: 7.5pt, отступ 6mm (было 4mm)
- Цитаты: 7.5pt (было 7), отступ 10mm (было 8mm), italic-стиль через `«»`
- Fact text (`→`): 7pt, отступ 10mm
- Увеличить зазор между источниками: 3mm + линия + 3mm (было 2+2)

### 7. Перераспределение контента
Уменьшить `SECTION_GAP` с 8 до 6mm для блока источников, чтобы они были компактнее и не создавали лишних пустых страниц.
