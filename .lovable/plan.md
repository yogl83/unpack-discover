

# План: Система управления партнерствами МИЭМ НИУ ВШЭ

## Что строим
Внутреннее веб-приложение на русском языке для управления индустриальными партнерствами МИЭМ НИУ ВШЭ. Система включает 10+ сущностей, 14 экранов, 3 роли доступа и Supabase-бэкенд.

## Архитектура

```text
┌─────────────┐     ┌──────────────┐     ┌────────────┐
│  React SPA  │────▶│ Supabase API │────▶│ PostgreSQL │
│  (Lovable)  │◀────│  + Auth + RLS│◀────│  10 tables │
└─────────────┘     └──────────────┘     └────────────┘
```

## Этапы реализации

### Этап 1. Инфраструктура и авторизация
- Подключить Supabase (через Lovable Cloud или внешний проект)
- Выполнить SQL-схему (`03_SUPABASE_SCHEMA.sql`) — 10 таблиц, 2 view, триггеры, индексы
- Настроить таблицу `user_roles` с RLS (admin / analyst / viewer) по рекомендованному паттерну с `has_role()` security definer
- Реализовать страницу входа (Supabase Auth, email/password)
- Контекст авторизации: хук `useAuth` + проверка роли

### Этап 2. Каркас приложения
- Левый сайдбар с навигацией (9 пунктов: Партнеры, Задачи партнеров, Гипотезы, Следующие шаги, Коллективы МИЭМ, Компетенции, Источники, Подтверждения, Пользователи)
- Layout с header (имя пользователя, роль, выход)
- Роутинг на 14 страниц
- Защита маршрутов по ролям

### Этап 3. CRUD-страницы — внешний контур
Для каждой сущности: список с фильтрами/поиском + карточка с формой + связанные секции.

1. **Партнеры** — список (фильтры: отрасль, статус, приоритет, город, владелец) + карточка (контакты, задачи, гипотезы, шаги, источники, подтверждения)
2. **Контакты** — inline в карточке партнера
3. **Источники** — список + создание (привязка к партнеру)
4. **Подтверждения** — список + создание (полиморфная привязка через entity_type/entity_id)
5. **Задачи партнеров** — список + карточка (описание, контекст, гипотезы, шаги)
6. **Гипотезы сотрудничества** — список + карточка (связь: партнер → задача → коллектив → компетенция)
7. **Следующие шаги** — список + карточка (полиморфная привязка к партнеру/задаче/гипотезе)

### Этап 4. CRUD-страницы — внутренний контур
8. **Коллективы МИЭМ** — список + карточка (компетенции, форматы, портфель)
9. **Компетенции** — список (привязка к коллективу)

### Этап 5. Пользователи и Dashboard
10. **Пользователи** — список (только admin может менять роли)
11. **Dashboard** — сводка: кол-во партнеров, задач, гипотез, шагов; просроченные шаги; недавняя активность

### Этап 6. Визуальная доработка
- Бейджи статусов и приоритетов (цветовые)
- Русские подписи всех полей, кнопок, пустых состояний, валидаций (по словарю `04_UI_MAPPING_RU_EN.md`)
- Компактный деловой UI

## Технические решения
- **Роли**: отдельная таблица `user_roles` + функция `has_role()` (security definer) + RLS-политики
- **Supabase-клиент**: единый файл `src/integrations/supabase/client.ts`
- **Типы**: автогенерация из схемы Supabase
- **Формы**: React Hook Form + Zod-валидация
- **Таблицы**: shadcn/ui Table с сортировкой и фильтрами
- **UI-словарь**: константы русских подписей из `04_UI_MAPPING`

## Первый шаг
Начнем с подключения Supabase и создания каркаса: layout, сайдбар, роутинг, авторизация, страница партнеров (список + карточка). Это даст рабочий прототип для итераций.
