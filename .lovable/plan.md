

# Схема профайла и посекционные промты в админке

## Проблема

Текущий единый промт даёт общие описания без конкретики. Эталонный профайл (Huawei) содержит:
- Нумерованные ссылки на источники `[1]`, `[2]`...
- Таблицы с конкретными цифрами
- Детальную аналитику по каждой секции
- Разный стиль для разных секций (business_scale — табличный, collaboration_opportunities — списки с обоснованием)

## Решение

Заменить единый промт на **схему профайла** — JSON-массив из 13 секций, где каждая секция имеет своё название, описание и индивидуальный промт. Плюс общий системный промт. Всё редактируется в админке.

## Что изменится

### 1. Структура настроек в `app_settings`

Вместо двух ключей (`ai_profile_model`, `ai_profile_prompt`) — три:

- `ai_profile_model` — без изменений
- `ai_profile_system_prompt` — общий системный промт (контекст МИЭМ, правила форматирования, требования к ссылкам)
- `ai_profile_sections` — JSON-массив со схемой секций:

```json
[
  {
    "key": "summary_short",
    "title": "Краткое описание",
    "prompt": "Напиши 2-3 предложения. Укажи основную деятельность, масштаб и уникальное ценностное предложение. Не повторяй информацию из других секций. Укажи источники в формате [N]."
  },
  {
    "key": "business_scale",
    "title": "Масштаб и показатели",
    "prompt": "Представь данные в формате таблицы: выручка, сотрудники, география, рейтинги. Укажи структуру бизнеса по сегментам с процентами. Каждый факт должен иметь ссылку [N]. Если точных данных нет — пиши 'Данные не найдены'."
  },
  ...
]
```

### 2. Обновлённый системный промт (дефолт)

Включит:
- Контекст МИЭМ (кратко)
- **Правила форматирования**: markdown, таблицы, нумерованные списки
- **Правила ссылок**: каждый факт сопровождается `[N]`, в конце секции — список источников
- **Антиповторы**: не дублировать факты между секциями
- **Требование конкретики**: цифры, даты, названия, а не общие слова

### 3. Обновлённая админка (`AdminAISettings.tsx`)

Три карточки:
1. **Модель** — dropdown (без изменений)
2. **Системный промт** — textarea для общих инструкций
3. **Схема секций** — аккордеон из 13 секций, каждая с:
   - Название (readonly)
   - Промт для секции (textarea)
   - Кнопки добавить/удалить секцию (на будущее)

Кнопки «Сохранить всё» и «Сбросить к умолчанию».

### 4. Обновлённая edge function

Вместо одного вызова AI с единым промтом:
- Загрузить схему секций из `app_settings`
- Собрать системный промт = общий промт + детальные инструкции по каждой секции из схемы
- Передать в tool calling с описаниями из промтов секций
- Добавить инструкцию по генерации `references` — массив `{number, text, url}` для каждого источника

### 5. Новое поле `references_json` (уже существует в БД)

Таблица `partner_profiles` уже имеет поле `references_json` (jsonb, default `[]`). Используем его для хранения ссылок в формате:
```json
[
  {"number": 1, "text": "Huawei — Википедия", "url": "https://ru.wikipedia.org/wiki/Huawei"},
  {"number": 2, "text": "О компании Huawei", "url": "https://www.huawei.ru/about/"}
]
```

## Файлы

| Файл | Действие |
|------|----------|
| `src/components/AdminAISettings.tsx` | Переписать: 3 карточки, аккордеон секций |
| `supabase/functions/generate-partner-profile/index.ts` | Читать схему секций, собирать составной промт, генерировать references |
| Миграция (данные) | Upsert `ai_profile_system_prompt` и `ai_profile_sections` в `app_settings` |

Изменений в структуре БД нет — таблица `app_settings` и поле `references_json` уже существуют.
