

# Улучшения диалога портфолио коллектива

## Замечания пользователя

1. **Тип предзаполнен, но селектор всё равно показан** — при добавлении из секции «Проекты» тип уже задан как `project`, но пользователю всё равно нужно видеть/менять его в форме. Решение: скрыть селектор типа, если элемент добавляется из конкретной категории, и показать тип в заголовке диалога.

2. **Поле «Название» без плейсхолдера** — нет подсказки в пустом поле. Добавить динамический placeholder в зависимости от типа: «Наименование проекта», «Название публикации» и т.д.

3. **Публикации и патенты из данных участников** — это задача на будущее (агрегация из внешних источников). Пока оставим ручной ввод, но запомним как фичу.

## Технические изменения

### UnitDetail.tsx — диалог портфолио

1. **Скрыть селектор типа** когда тип предзаполнен из accordion-секции. Показать тип текстом в заголовке диалога: «Новый проект», «Новая публикация» и т.д.

2. **Добавить placeholder** в поле Название, зависящий от `pForm.item_type`:
   - project → «Наименование проекта»
   - publication → «Название публикации»
   - patent → «Название патента»
   - grant → «Название гранта»
   - product → «Название продукта»
   - other → «Название»

3. **Заголовок диалога** — при создании показывать тип: «Новый проект» / «Новая публикация» / «Редактировать проект» вместо обезличенного «Новый элемент портфолио».

### Затронутый файл
- `src/pages/UnitDetail.tsx` — только изменение диалога
