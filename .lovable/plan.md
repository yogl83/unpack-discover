

# Улучшение ссылок на источники: от общих URL к проверяемым цитатам

## Проблема

Сейчас ссылки на источники — это просто номера `[N]`, которые ведут на общий список URL внизу. Читатель не может быстро проверить конкретный факт, потому что:
- Ссылка ведёт на главную страницу сайта или на результат поиска
- Нет цитаты из источника, подтверждающей факт
- Нет указания, на какой именно странице или в каком абзаце найдена информация

Пример: факт «участие в мероприятиях с численностью до 2000 человек» ссылается на источник, но источник — это страница disclosure.ru с совсем другой информацией.

## Решение

### 1. Собирать цитаты-подтверждения при генерации

При генерации профайла AI уже получает полные тексты источников. Нужно потребовать, чтобы для каждой ссылки `[N]` AI также возвращал **короткую цитату** из источника, подтверждающую факт.

Изменение в промте и tool schema:
- Добавить в `references` поле `quote` — дословная цитата из источника (1-2 предложения)
- Добавить поле `page_context` — раздел/заголовок страницы, где найден факт

Формат references станет:
```json
[
  { "number": 1, "text": "Сайт компании", "url": "https://...", 
    "quotes": [
      { "fact_text": "выручка 5 млрд руб.", "source_quote": "Выручка группы в 2024 г. составила 5,1 млрд руб." }
    ]
  }
]
```

### 2. Показывать цитаты при наведении на [N]

При клике/наведении на ссылку `[N]` в тексте — показывать всплывающее окно (Popover) с:
- Название источника + кликабельная ссылка
- Цитата из источника, подтверждающая факт
- Если цитаты нет — пометка «Прямая цитата не найдена»

### 3. Расширить блок «Источники» внизу

Для каждого источника показывать не только название+URL, но и список цитат, использованных из него. Это позволяет читателю видеть, какие именно данные были извлечены из каждого источника.

### 4. Факт-чекинг: показывать explanation

Результаты факт-чекинга уже содержат `explanation` для каждого факта. Нужно показывать его в UI рядом с отметками ⚠️/❌, чтобы читатель видел, почему факт помечен как неподтверждённый.

## Файлы

| Файл | Изменение |
|------|-----------|
| `supabase/functions/generate-partner-profile/index.ts` | Обновить промт и tool schema: требовать `quotes` в references; обновить `buildReferencesFromSources` для передачи контекста |
| `src/components/partner/PartnerProfileTab.tsx` | Заменить простой скролл `[N]` на Popover с цитатой; расширить `ReferencesBlock` цитатами; показывать `explanation` в результатах факт-чекинга |

## Результат

Читатель сможет:
- Навести на `[3]` в тексте → увидеть «Источник: disclosure.ru — *"Выручка группы составила 5,1 млрд руб."*»
- Понять, что факт подтверждён конкретной цитатой, а не абстрактной ссылкой
- Если цитаты нет — сразу увидеть, что факт может быть ненадёжным
